# MOD_7_EVALUACION

M7_Evaluación del módulo[Actividad Evaluada]

Evaluación del módulo
Esta actividad tiene como objetivo que crees y presentes un portafolio de productos que refleje tu aprendizaje y habilidades adquiridas a lo largo del curso. Este portafolio servirá como una herramienta clave para mostrar tus proyectos, desarrollos y capacidades técnicas.

Tu tarea es realizar los siguientes pasos:

1. Creación del Portafolio de Productos:

Portafolio en GitHub: Debes crear un repositorio en GitHub que contenga tu portafolio de productos. Asegúrate de incluir:

Una página de perfil de GitHub bien configurada.

Repositorios organizados de tus proyectos, cada uno con una buena página principal, una descripción clara y las instrucciones de uso si es necesario.

Buenas prácticas como README detallados, documentación clara y etiquetado adecuado.

Portafolio en Behance: Si prefieres, puedes utilizar Behance como plataforma para presentar tu portafolio, especialmente si has trabajado en proyectos de UX/UI:

Crea un proyecto dedicado en Behance, destacando tus trabajos más relevantes.

Asegúrate de que la presentación sea visualmente atractiva, clara y profesional.

2. Alojamiento de tus Proyectos:

Elige al menos un proyecto para alojarlo en un servidor gratuito o plataforma de hosting. Algunos ejemplos son:

GitHub Pages (para proyectos front-end).

Render: (Plataforma gratuita y fácil de usar para desplegar aplicaciones Java con base de datos PostgreSQL.).

Railway (Ideal para proyectos backend con Spring Boot. Ofrece despliegue automático desde GitHub y soporte para PostgreSQL.).

Vercel o Netlify: Aunque están orientadas a frontend, puedes usarlas para alojar el cliente web de un proyecto full stack (por ejemplo, una SPA que consume tu API de Spring Boot).
Asegúrate de que los proyectos estén disponibles en línea, funcionando correctamente y sean accesibles desde un dominio público.

3. Creación de un Video Tutorial Técnico:

Graba un video en YouTube donde expliques y presentes tu producto final. En el video debes cubrir:

Una breve introducción sobre el proyecto.

La tecnología utilizada.

Demostración de funcionalidades clave del producto.

Cualquier desafío que hayas enfrentado y cómo lo solucionaste.

Asegúrate de que el video sea claro, bien estructurado, y tenga una duración adecuada (de 5 a 10 minutos). Utiliza herramientas gratuitas como OBS Studio o Loom para grabar.

4. Feedback y Retroalimentación:

Invita a otros compañeros o colegas a revisar tu portafolio y proyectos. Recibe retroalimentación y realiza las mejoras necesarias en base a sus comentarios.

Refleja esta retroalimentación en tu proyecto y asegúrate de depurar y mejorar la presentación y funcionalidad del producto final.

5. Entrega Final:

Subir tu portafolio y proyecto a GitHub o Behance (dependiendo de tu elección).

Asegúrate de que el enlace a tu portafolio en línea esté accesible y funcional.

Subir el enlace al video tutorial en YouTube junto con los enlaces a tus proyectos alojados.

Presenta un archivo README con el resumen de tus proyectos y el proceso de creación, los retos encontrados y cómo los solucionaste.

Requisitos

Portafolio Atractivo: Asegúrate de que tu portafolio en GitHub o Behance sea profesional y organizado.

Alojamiento: Debes alojar al menos un proyecto en línea utilizando un servicio gratuito de hosting.

Video Tutorial: Presenta un video en YouTube que demuestre tus habilidades y explique tu producto.

Feedback: Aplica la retroalimentación de tus compañeros para mejorar tus proyectos.

Entregables: Incluye el enlace a tu portafolio, el video de YouTube y la documentación completa de tus proyectos.
